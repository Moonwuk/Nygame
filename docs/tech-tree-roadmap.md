# Дерево технологий — технический roadmap

> Расширяет **уже существующее** сессионное дерево технологий (✅ `TechnologyDef` +
> `technology`-модуль + `tech.research` + `data/technologies.json`, backlog Блок C/C1):
> добавляет **4 ветки** (наземная / космическая / эскадрилья / ракеты), **день-гейт**
> (технология открывается с определённого дня) и **условия** открытия. Сессионное
> (сбрасывается с матчем, GDD §5.1), детерминированное, читается снапшотом при старте
> (GDD §5.2). Дерево — **бэкбон анлоков** всего контента (юниты/здания/модули по ветке).
> Парные доки: `technology`-модуль/`schemas.ts` (TechnologyDef), `gdd.md` §5,
> `account-level.md` (мета: какие узлы доступны), `squadrons`/`missiles`/`shields`-роадмапы
> (что ветки открывают), `mechanics-roadmap.md`. Формат — кирпичики. Зоны: `[core]`·`[data]`·`[cli]`.

---

## 0. Что уже есть и что добавляем (read first)

**✅ База (в коде):** `TechnologyDef` = `tier`, `cost`, `researchTimeHours`, `prerequisites[]`,
`unlocks{units,buildings,abilities}`, `effects{productionBonus,fleetSpeedBonus,combatDamageBonus}`;
`tech.research`-действие → отложенное завершение (`schedule`) → анлоки/эффекты; `completed[]`/`active`
на игроке; снапшот при старте.

**✅ Уже добавлено (в коде):**
- **4 ветки** (`branch`): `ground` · `space` · `squadron` · `missile` — вкладки UI = ветки.
  ✅ уже в коде: `schemas.ts` (`branch` enum, default `space`), `data/technologies.json` (узлы несут ветку).
- **День-гейт** (`dayGate`): технология **доступна к исследованию только с дня N** сессии
  (мировое время, timeScale-масштаб) — пейсинг эпох, нельзя зарашить всё в начале.
  ✅ уже в коде: `schemas.ts` (`dayGate`), `technology.ts` (гейт → `E_TOO_EARLY`).
  ✅ контент проставлен (контент-пас эпох): гейты `2/3/5/8/12/15` в обоих бандлах
  (`data/technologies.json` + инлайн-бандл прототипа). Гейт **жёсткий** (решение
  владельца 2026-07-17); «мягкий» (можно раньше, но дороже) — потенциальный data-флаг
  follow-up, ядро под него не трогали. UI-подпись «День N» = `dayGate+1` (счёт
  статус-бара, день 1 — первый).

**⏳ Этот роадмап ещё добавляет:**
- **Условия** (`conditions[]`): открытие при выполнении игровых условий (сверх prereq-техов).
- **Правило открытия:** техно доступно, когда **день ≥ `dayGate` И prereqs завершены И
  `conditions` выполнены**. Затем исследуешь (cost + время → завершение → эффекты).

## Фаза 1 · Гейтинг и исследование `[core]`

### TT-1.1 · Правило доступности `[core]` 🔒(TT-0.*) — M
**Подзадачи:** техно **researchable**, когда `день ≥ dayGate` **И** все `prerequisites` в `completed`
**И** все `conditions` истинны. `tech.research` **отклоняет** недоступное (fail-secure стабильный код).
Детерминированно (день из `state.time`, условия из стейта).
**Готово, когда:** ранний/без-условий/без-prereq запуск исследования отвергается; доступное — стартует; тест.

### TT-1.2 · Исследование → завершение (reuse) `[core]` 🔒(TT-1.1) — S
**Подзадачи:** переиспользовать существующее: отложенное завершение (`schedule(researchTimeHours)`) →
`unlocks`/`effects` (capabilities/хуки). Убедиться, что гейтинг TT-1.1 соблюдается на старте.
**Готово, когда:** доступное техно исследуется за время и даёт анлоки; повтор не дублирует (idempotent).

### TT-1.3 · Слоты исследований (РЕШЕНО: 2 → до 3) `[core]` 🔒(TT-1.1) — S
**Решение:** **N одновременных слотов под любую ветку** (не «по ветке», не «одно глобально»):
**2 базовых слота**, с возможностью **увеличить до 3** (макс). `active` становится **списком** (≤ N),
каждый слот — любое доступное техно из любой ветки.
**Источник +1 слота (РЕШЕНО):** даёт **учёный-«+слот»** (см. Фаза 4) — его профиль вместо
ветко-бонуса/капстоуна другого учёного (opportunity-cost). **НЕ % ускорения**, не донат-эксклюзив.
**Готово, когда:** одновременно идёт ≤ N исследований; апгрейд поднимает лимит до 3; тест.

## Фаза 2 · Мета-слой (какие узлы вообще доступны) `[srv][core]`

**Готово, когда:** недоступный по уровню узел не появляется в дереве матча; снапшот иммутабелен.

## Фаза 4 · Учёный (research-лидер) `[data][core]`

> **Учёный — выбираемая мета-сущность (НЕ юнит, не боевой герой):** research-лидер, выбирается **в
> самом начале** (консервация/снапшот, GDD §2/§5.2), как стартовый профиль. **Уровень — мета**
> (качается между сессиями, `account-level`, earnable F2P; заносится снапшотом). Даёт **качественные**
> бонусы (фокус ветки / доп. слот / поздний капстоун), **НЕ % к ускорению** — сознательно (не вертикаль, не %-гонка).

### TT-4.1 · Слот учёного + выбор на старте `[data][core]` 🔒(TT-0.1) — M
**Подзадачи:** **учёный — сущность выбора, не `UnitDef`** (отдельный каталог `scientists`, данные);
выбор при создании матча (снапшот, иммутабельно); **тип** (1 из 4 веток) + **мета-уровень** (из
`account-level`, earnable) → **бонус к изучению своей ветки** (фокус, не общий %). Не в `hero`-модуле
(он про боевых героев) — отдельный слот.
**Готово, когда:** учёный выбирается на старте; его ветка исследуется с бонусом; уровень из снапшота аккаунта; иммутабельно.

### TT-4.2 · Капстоун: супер-юнит / особое здание (лейт-гейм) `[data][core]` 🔒(TT-4.1) — M
**Подзадачи:** **макс-уровневый** учёный **открывает супер-юнит или особое здание** — **горизонтальный
анлок** (новая способность, не «+%»). **Чисто лейт-гейм:** капстоун-контент **день-гейтится поздно**
(высокий `dayGate`) — не ранний раш, а эндгейм-награда (баланс через позднее окно + контр-игра, как
ядерки). **P2W-guardrail:** мета-уровень учёного **earnable игрой** (F2P), не донат-эксклюзив.
**Готово, когда:** макс-учёный открывает капстоун **только в лейте**; он контрится; earnable F2P; тест.

### TT-4.3 · Учёный «+слот» `[data][core]` 🔒(TT-4.1, TT-1.3) — S
**Подзадачи:** один **тип учёного даёт +1 слот исследования** (2→3) вместо ветко-бонуса/капстоуна —
opportunity-cost. **Явно: НЕ даёт % к ускорению исследований.**
**Готово, когда:** выбор этого учёного поднимает слоты до 3; % ускорения он не даёт; тест.

## Инварианты (нарушение = баг)

- **Детерминизм:** день из `state.time`, условия из стейта, завершение через `schedule`; нет
  `Date.now`/`Math.random`.
- **Data-driven дерево + курируемые условия** (готовые блоки, не пользовательский конструктор, §7.5).
- **Сессионное + снапшот при старте** (GDD §5.1/§5.2): дерево сбрасывается с матчем; набор узлов
  фиксируется на старте.
- **Server-authority + fail-secure:** `tech.research` гейтится сервером; недоступное → стабильный код.
