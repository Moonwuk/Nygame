# Security Master Plan — «самое безопасное приложение и игра»

Зонтичный план над тремя техническими треками безопасности проекта. Он не
дублирует их, а **синтезирует и приоритизирует**: даёт честную оценку текущей
посадки, единый north star, сквозную модель угроз (приложение **и** игра) и
последовательность действий к «безопасно по умолчанию». Новые предложения,
которых ещё нет в под-роадмапах, оформлены как бирки `MP-*`.

Под-треки (детальные задачи живут там, здесь — только сшивка и приоритет):

- `secure-sdlc-roadmap.md` — как мы **строим** безопасно (бирки `SD-*`).
- `secure-environment-roadmap.md` — как мы безопасно **эксплуатируем** (`SE-*`).
- `game-integrity-roadmap.md` — античит/честность игры (`GI-*`).
- `security-a06.md` — insecure design / злоупотребления механиками (`SEC-A06-*`).
- `https-roadmap.md` — перевод на TLS (`HTTPS-*`).
- `backlog.md` «Блок SEC» — плоские бирки пайплайна (`SEC-*`), `docs/security/pipeline.md` — база.

Легенда статусов (как во всём репо): ✅ сделано · ⏳ можно брать · 🔒(dep) ждёт
зависимость. Соответствие бирок: `SEC-1→SD-0.2`, `SEC-2→SD-2.1`, `SEC-3→SD-6.1`,
`SEC-4→SD-6.3`, `SEC-5/7→SD-3.3`, `SEC-6→SD-8.2`, `SEC-8→SD-8.3`; `F5→SE-6.3`,
`F6→SE-6.4`, `F7→SE-0.1`.

## 1. North star и принципы

**«Безопасное — единственно возможное».** Цель — не «есть защита», а: в проде
**нельзя** запустить небезопасную конфигурацию; защита на каждом слое (zero-trust
между тирами); game-integrity — полноправный домен безопасности; **детекция и
реакция** наравне с профилактикой.

Принципы (наследуют инварианты `CLAUDE.md` и anti-goals под-роадмапов):

1. **Server-authority.** Клиент шлёт намерение, не состояние. Клиентский античит —
   не граница доверия.
2. **Fail-secure (OWASP A10).** Любая ошибка → типизированный отказ `E_*`, без утечки
   деталей. Сканер, который не отработал, ничего не сертифицирует.
3. **Secure-by-default.** Небезопасный режим не запускается в проде — не «нельзя по
   договорённости», а «нельзя технически».
4. **Defense-in-depth.** Дублирующие контроли на разных слоях/движках; падение одного
   не открывает систему.
5. **Детерминизм как контроль целостности.** Воспроизводимость сима — не только фича,
   но и инструмент обнаружения жульничества (см. `MP-2`).
6. **Минимум PII.** Особенно в сигналах анти-мультиаккаунта и в логах.

## 2. Текущая посадка (честно, сверено с кодом)

**Уже сильно (✅):** fail-secure ядро; server-authority; `visibleState` + fog-on-send
(`F6`/`SE-6.4`); action-gate `@void/action-layer` (envelope→payload→authz→dedup→
sequence, `GATE=1`); идемпотентные bounded rate-limited квитанции; commit-before-
broadcast над Postgres (`F2`/`F8`); JWT-handshake + Origin-allowlist (`F7`/`SE-0.1`);
scrypt-аккаунты с униформным 401 и decoy-таймингом + per-IP rate-limit; детерминизм с
golden-RNG и **replay-проверкой** (`RPL-1..4`), property/fuzz-тесты (`FUZZ-1..4`); CI —
8 сканеров с ratcheting-гейтом (`SEC-0/1`), кастомные Semgrep-правила инвариантов
(`SEC-2`), SARIF→Code Scanning (`SEC-4`), SHA-пины + digest-пины + SBOM + cosign на APK
(`SEC-3/5/7`); анти-сговор `E_BOT_ALLIANCE` (коалиции — только люди); A06-контроли
(opaque reject-коды `SEC-A06-1`, cap ордеров `SEC-A06-2`).

**Разрыв — «построено ≠ включено».** Auth, gate, TLS, `SEAT_LOCK` — **opt-in / off по
умолчанию**. Реально играбельный путь (`pnpm host` / `prototype/netserver`) пускает по
нику **без пароля** и принимает bare-actions (readiness-gap #1). Это — приоритет №1.

**Не построено (профиль детекции/инфры):** анти-cheat rate-shaping и аномалия-детект
(`GI-0.1`,`GI-1.x` 🔴); анти-мультиаккаунт/боты (`GI-2.x` 🔒); durable action-log для
audit-replay/отката (`RPL-5`/`PE-1.1` 🔒 — квитанции пишутся без type/payload/time);
секрет-стор/ротация (`SE-2.x`); БД-хардненинг/бэкапы/DR (`SE-3.x`,`SE-9.x`); алертинг
(`SE-8.x`); OIDC-identity; multi-process scale (pg-boss); CSP/Trusted Types (`SE-7.x`);
TLS на self-host (`HTTPS-2.x/5.x`).

## 3. Сквозная модель угроз (домены)

| Домен | Ключевые угрозы | Где закрывается |
| --- | --- | --- |
| **Транспорт/периметр** | MITM, CSWSH, DDoS, раскрытие origin | `HTTPS-*`, `SE-1.*`, `MP-1` |
| **AuthN/AuthZ/сессии** | slot-hijack, кража токена, брутфорс, replay | `F7`/`SE-0.1`, `SE-0.2`, `MP-8` |
| **Инъекции/вывод** | XSS (DOM/UGC), SQLi, prototype-pollution, ReDoS | `SD-1.*`, `MP-5` |
| **Supply-chain** | компрометация npm/actions/образов/**game-data** | `SEC-3/5/7`, `SD-3.*`, `MP-4` |
| **Секреты/инфра** | утечка ключей, публичная БД/Redis, отсутствие бэкапов | `SE-2.*`,`SE-3.*`,`SE-4.*`,`SE-9.*` |
| **Game-integrity** | maphack, double-spend, дюп экономики, боты, мультиакк, сговор | `GI-*`, `SEC-A06-*`, `MP-2/3` |
| **Наблюдаемость/реакция** | нет детекции аномалий, нечем расследовать/откатить | `SE-8.*`, `GI-1.*`, `RPL-5`, `MP-7` |
| **Приватность** | избыток PII, нет export/erasure | `SE-10.1`, `MP-6/7` |

## 4. Приоритетная дорожная карта

### Tier 0 — Закрыть разрыв «capability vs enforcement» (до любого публичного доступа)

Максимальный ROI: делает уже построенное **неотключаемым**.

| Действие | Бирки | Готово, когда |
| --- | --- | --- |
| Secure-by-default launch guard | **`MP-1`** | `PROD=1` fail-closed без auth+gate+TLS+seat-lock |
| Auth+gate обязательны на играбельном пути | `SES-2.5`, readiness#1 | nick-only handshake недоступен в проде; bare-action всегда отклоняется |
| TLS перед сервером + запрет non-https | `HTTPS-2.1/1.1`, `TLS-4/5` | Caddy+ACME; `x-forwarded-proto≠https`→reject; release-APK без cleartext (`HTTPS-5.1`) |

### Tier 1 — Периметр и секреты (по мере посадки инфры `F1/F2`)

Cloudflare WAF/DDoS + скрытие origin (`SE-1.1/1.3`), TLS 1.3 в приватной сети
(`SE-1.2`), egress-allowlist (анти-SSRF). Секрет-стор + ротация (`SE-2.1/2.2`),
ротация ключей JWT/JWKS (остаток `SE-0.1`), refresh-token rotation с reuse-detection
(**`MP-8`**). БД least-priv + TLS verify-full + RLS + шифрование at-rest + **бэкапы/PITR
с протестированным restore** (`SE-3.1..3.4`).

### Tier 2 — Runtime-abuse и game-integrity (сердце «самой безопасной игры»)

| Действие | Бирки | Приоритет/заметка |
| --- | --- | --- |
| Durable append-only action-log | `RPL-5`/`PE-1.1` | **фундамент детекции** — разблокирует audit-replay `GI-1.3`, откат, санкции `GI-3.1` |
| Shadow re-simulation из лога | **`MP-2`** | сравнение `hashState` независимого воркера ловит вмешательство |
| Economy-conservation как инвариант | **`MP-3`** | property-тест: сохранение ресурсов ловит дюпы генерически (`GI-2.3`) |
| Rate-shaping + per-player queue на масштабе | `GI-0.1/0.2`, `SE-6.2/6.3` | cross-process optimistic locking |
| Fairness-метрики + аномалия-алерты | `GI-1.1/1.2`, `SE-8.x` | всплеск reject/authz/rate-limit → детект (A09) |
| Анти-мультиаккаунт/сговор + бот-детект | `GI-2.1/2.2` | нужен identity (`AC-0.1`) + minimal-PII (`SE-10`) |
| Turnstile/PoW на регистрации | **`MP-6`** | дешёвый ранний анти-Sybil до поведенческих систем |

### Tier 3 — Клиент и приватность

CSP + Trusted Types + HSTS + SRI (`SE-7.1/7.2`, `SD-1.2`) + `eslint-plugin-no-unsanitized`
(`SD-2.2`) — **структурное устранение всего XSS-класса** (`MP-5`): DOM-XSS находки
Code Scanning при Trusted Types невозможны by construction. Новый клиент
(`packages/client`) — рождать сразу XSS-непробиваемым, не таща ручную защиту legacy-
`prototype/`. GDPR/data-minimization (`SE-10.1`, **`MP-7`**): export/erasure, PII-firewall.

### Tier 4 — Процесс и assurance

Blocking-сканеры как **required checks** на `main` (остаток `SD-0.2`) + CODEOWNERS +
signed commits (`SD-5.1/5.2`). Threat-model-as-code + пер-фича misuse-gate (`SD-4.1/4.2`,
`SEC-A06-10`). DAST/ZAP (`SEC-6`) после живого сервера. Полный OWASP Top-10 + ASVS L2
(`SEC-8`, `SD-8.1`), vuln-response SLA (`SD-8.3` — засеян `SECURITY.md`). IR/DR:
runbooks, RTO/RPO, kill-switch (`SE-9.2`).

## 5. Новые бирки (`MP-*`) — то, чего ещё нет в под-роадмапах

Формат: `id · зона · статус · «Готово, когда»`. Каждая расширяет существующий трек.

- **MP-1** `[srv][sec]` ✅ — **Secure-by-default launch guard.** `PROD=1`/`PROD=true` —
  явный триггер, dev-харнес (`PROD` не задан) не тронут ни на йоту (`pnpm dev:server`
  как был). `checkProductionReadiness` (`serverConfig.ts`, чистая функция от `env`) при
  `PROD` проверяет `AUTH_JWT_SECRET` + `GATE=1` + TLS (`TLS_KEY_FILE`+`TLS_CERT_FILE`
  нативно, ИЛИ `TRUST_PROXY=1` за терминирующим прокси) + `SEAT_LOCK=1` — `main.ts` при
  `!ok` пишет, чего не хватает, и `process.exit(1)` ДО поднятия сторов/сети (fail-closed).
  Попутно найден и закрыт реальный пробел: `SEAT_LOCK` был реализован только в
  прототипном `netserver.ts` — на боевом входе (`main.ts`) опция `seatLock` у
  `createMultiplayerServer` вообще не была подключена (проверка проверяла бы флаг без
  эффекта). Портирован тот же паттерн, что в `netserver.ts` + строка статуса `seats` в
  boot-логе. Тесты: `serverConfig.test.ts` (все ветки missing/ok, `'1'`/`'true'` формы,
  native-TLS vs TRUST_PROXY). Прогнано вживую: dev без `PROD` — стартует как раньше;
  `PROD=1` без секретов — `exit 1` с точным списком; `PROD=1` + все четыре переключателя
  — стартует, `/health` отвечает, seat-lock включён.
- **MP-2** `[srv][sec]` 🔒(`RPL-5`) — **Shadow re-simulation.** Периодически пере-
  симулировать матч из durable action-log на независимом воркере и сравнивать
  `hashState`. Расхождение = вмешательство/эксплойт/десинк → сигнал `GI-1.x`.
  _Готово, когда:_ фоновой ре-сим гоняет N матчей, дивергенция поднимает алерт + дамп.
- **MP-3** `[core][sec]` ⏳ — **Economy-conservation property.** fast-check свойство: на
  любой последовательности действий суммарные ресурсы сохраняются, кроме явных
  source/sink. Генерический детектор дюпов (дополняет `GI-2.3` как стоячее свойство).
  _Готово, когда:_ свойство в CI зелёное и ловит подсаженный дюп в тесте-негативе.
- **MP-4** `[data][sec]` ⏳ — **Подпись/верификация game-data бандлов.** `data/*.json`
  задаёт правила сима; подмена бандла меняет правила. Композированный бандл хэшируется
  (и опц. подписывается), хэш верифицируется при загрузке; матч уже пинит версию данных
  — добавить проверку целостности. _Готово, когда:_ загрузка отклоняет бандл с неверным
  хэшем; хэш пинится в манифесте матча.
- **MP-5** `[cli][sec]` 🔒(`CP0.1`) — **Trusted Types-first клиент.** В `packages/client`
  включить Trusted Types + CSP + `no-unsanitized` с рождения, чтобы DOM-XSS-класс был
  структурно невозможен. _Готово, когда:_ CSP с `require-trusted-types-for 'script'`;
  lint запрещает сырой `innerHTML`; e2e без CSP-violation.
- **MP-6** `[srv][sec]` 🔒(accounts) — **Anti-automation на регистрации/логине.**
  Turnstile/PoW поверх per-IP rate-limit, чтобы удорожить массовое создание аккаунтов
  до появления поведенческих систем (`GI-2.1`). _Готово, когда:_ регистрация требует
  проверки; e2e на прохождение/отказ.
- **MP-7** `[sec][ops]` 🔒(`SE-8.1`) — **PII-firewall для логов.** Типизированный слой
  редакции + CI-тест, гоняющий секрет-паттерны (Gitleaks/TruffleHog) по **runtime-логам**,
  а не только по репо. Правило «логи без PII/секретов» — в проверяемый инвариант.
  _Готово, когда:_ тест краснеет на подсаженный токен в логе; редакция активна по
  умолчанию.
- **MP-8** `[srv][sec]` 🔒(`SE-0.1`) — **Refresh-token rotation с reuse-detection.**
  Ротация refresh-токенов сессии с обнаружением повторного использования (украденный
  токен → инвалидция семьи). _Готово, когда:_ повторное предъявление отозванного
  refresh-токена рвёт всю сессионную семью; тест на reuse.

## 6. Definition of «secure-enough to launch» (гейт публичного запуска)

Публичный запуск НЕ открывается, пока не выполнено (сшито с `roadmap.md` «Этап 7»):

- [x] `MP-1` — прод не стартует без auth+gate+TLS+seat-lock.
- [ ] Auth+gate включены на играбельном пути; bare-action отклоняется (readiness#1).
- [ ] TLS везде, где сервер открыт; release-APK без cleartext (`HTTPS-*`, `F-05`).
- [ ] Периметр: WAF/DDoS + скрытый origin (`SE-1.*`).
- [ ] Секреты в сторе, не в образе/репо; ключи ротируются (`SE-2.*`, `MP-8`).
- [ ] БД: least-priv + шифрование + **протестированный restore** (`SE-3.*`).
- [ ] Детекция: fairness-метрики + аномалия-алерты (`GI-1.*`, `SE-8.*`).
- [ ] Реакция: durable action-log + audit-replay + kill-switch (`RPL-5`,`GI-1.3`,`SE-9.2`).
- [ ] Клиент: CSP/Trusted Types/HSTS (`SE-7.*`, `MP-5`).
- [ ] Assurance: blocking-сканеры required на `main`; полный OWASP Top-10/ASVS L2
      (`SEC-8`, `SD-8.1`); vuln-response SLA (`SD-8.3`).

## 7. Последовательность (что раньше)

1. **Tier 0** целиком — самый дешёвый и самый важный (`MP-1` + enforcement + TLS).
2. `RPL-5` durable action-log — разблокирует весь блок детекции/отката/`MP-2`.
3. `MP-3` economy-conservation + `MP-5` Trusted Types — дешёвые структурные защиты.
4. Required checks + CODEOWNERS + signed commits (Tier 4 process).
5. Периметр/секреты/БД (Tier 1) — по мере посадки `F1/F2`.
6. Детекция и анти-абуз (`GI-1.x/2.x`, `SE-8.x`) → OIDC/anti-multiaccount.
7. DAST + полный OWASP-проход (`SEC-6/8`) как финальная закалка.

## 8. Anti-goals (подтверждаем под-роадмапы)

Без клиентского античита как границы доверия; без twitch-grade анти-чита; без
агрессивного бана без ревью (ложный бан хуже читера); без «временно» публичных
БД/Redis/plain-port; без секретов в образе/репо; без самопального крипто/TLS; без
security-theater (подпись образов там, где нет реального деплоя); без блокировки
сканеров на легаси-шуме.
