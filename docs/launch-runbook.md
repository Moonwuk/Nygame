# Запуск сервера и приглашение игроков — ранбук

> Проверено против кода и живьём (см. «Что проверено» внизу). Канонический путь
> для живых игроков сегодня — **прототип-хост** (`pnpm host`): единый HTML-клиент
> (браузер + APK) поверх авторитетного сервера с полным игровым ядром прототипа
> (27 модулей: дивизии, герои, дипломатия, биржа, шпионаж, «Хранитель», мета-прогрессия, победа).
> Альтернативный стек `packages/server` + `packages/client` — боевая
> инфраструктура (JWT-аккаунты, action-gate, мульти-матчи), но его клиент пока
> отдаёт только движение флота — игроков туда не приглашаем (см. «Дальше»).

## 1. Быстрый запуск (VPS / любая Linux-машина)

```bash
git clone https://github.com/Moonwuk/Nygame && cd Nygame
pnpm install

# (опционально, но нужно для «матч переживает рестарт») — Postgres:
cd deploy && POSTGRES_PASSWORD=<пароль> docker compose up -d && cd ..
#   docker-compose.yml биндит Postgres ТОЛЬКО на 127.0.0.1 — наружу не торчит.
#   Без Docker подойдёт любой локальный Postgres: создать БД/роль void.

cp deploy/server.env.example deploy/server.env   # PORT / TIME_SCALE / DATABASE_URL
bash deploy/serve.sh   # pull → install → (ре)старт в tmux-сессии «void» + health-poll
```

`serve.sh` сам перезапускает старую tmux-сессию, ждёт `/health` и печатает
ссылку для игроков. Логи: `tmux attach -t void`.

Ручной эквивалент (то, что делает `pnpm host`):

```bash
DATABASE_URL=postgres://void:<пароль>@127.0.0.1:5432/void \
TIME_SCALE=200 HOST=0.0.0.0 PORT=8788 pnpm host
```

Переменные (читает `prototype/netserver.ts`):

| Переменная | Значение |
| --- | --- |
| `HOST` / `PORT` | куда биндиться (для игроков — `0.0.0.0`; дефолт порт 8788) |
| `TIME_SCALE` | ускорение мира: 1 = реальное время (боевой режим), 200 = плейтест (реальная минута ≈ 3.3 игрового часа) |
| `DATABASE_URL` | Postgres → матч, receipts и закреплённые за никами места переживают рестарт; без неё — in-memory (рестарт теряет матч) |

## 2. Доступ игроков снаружи

Сервер отдаёт игру сам: игрок просто открывает `http://<адрес>:8788/`.

- **Та же сеть (LAN)** — ссылка с IP машины, больше ничего не нужно.
- **Интернет без своего домена** — туннель: `cloudflared tunnel --url http://localhost:8788`
  (даёт публичный `https://…trycloudflare.com`, WSS в комплекте). Диагностика сети: `pnpm doctor`.
- **Свой домен** — реверс-прокси с TLS перед портом 8788 (Caddy/Nginx).
  Готового `deploy/Caddyfile` ещё нет — кирпич HTTPS-2.1 в `docs/https-roadmap.md`.
- **Клиент из веба (GitHub Pages)** — workflow `pages.yml` задуман публиковать HTML на
  `https://moonwuk.github.io/Nygame/`, но **деплой не проходит**: все раны падают за
  ~3 с без раннера (окружение `github-pages` недоступно — включить Pages в
  Settings → Pages, для приватного репозитория нужен платный план). Пока клиента
  из веба нет — но он и не обязателен: сервер раздаёт игру сам на `/`.
- **Android APK** — workflow `android.yml`: артефакт на каждом ране + rolling-релиз
  `alpha` со стабильной ссылкой (сборки с `main`); в APK встроен автоапдейтер,
  сверяющий versionCode с релизом. В приложении игрок вводит адрес сервера на
  экране подключения.

## 3. Как выглядит вход для игрока

1. Открыл ссылку → «Вход по позывному» → хаб → **ИГРАТЬ СЕЙЧАС**.
2. Адрес сервера уже подставлен (same-origin), ввёл позывной → «Обновить список».
3. Вкладка «Доступные» → войти в свободное место. Лобби нет (SES-2.1, модель Iron
   Order): мир живёт 24/7 с момента создания сессии, игрок всегда попадает в уже идущую
   партию — ни хоста, ни кнопки Start.
4. Позывной = аккаунт: место закрепляется за ником (таблица `seats`), выход/возврат
   и рестарт сервера сохраняют место и мир. Пока игрок офлайн, за него держит
   оборону серверный AI (а с техом «Хранитель» — делегирование по желанию).

## 4. Ограничения текущего пути (важно при приглашении)

- **Несколько матчей в одном процессе:** `MATCHES` (1..16, дефолт 1) регистрирует комнаты
  `proto`, `proto-2`, … в одном `MatchRegistry` на **одной** БД (id'ы различны — конфликта
  нет), все видны в браузере матчей. Кресел на матч — до 10: по умолчанию FFA на 10,
  `TEAMS=5v5` использует все 10, `TEAMS=2v2` — компактный режим на 4.
- **Аккаунты подключены** (SES-2.5) и включаются переменной `AUTH_JWT_SECRET`: тогда клиент
  показывает поле пароля, регистрирует/логинит (`POST /auth/register|login` → session-JWT)
  и меняет сессию на join-токен (`GET /matches/:id/join`) для WS-хендшейка. Без
  `AUTH_JWT_SECRET` — dev-режим `?nick=` (кто знает ник, займёт место — для дружеского
  плейтеста ок) либо `SEAT_LOCK=1` (посадочные билеты, дефолт в compose).
- **TLS сам сервер не терминирует** — снаружи только через туннель/прокси.
- **Action-gate включается переменной `GATE`** (дефолт в `docker-compose` = `1`, ВКЛ):
  `GATE=1` ставит `ActionGate` перед редьюсером на netserver-пути — код менять не надо (REL-4).
- Конец матча: победа фиксируется и матч уходит в архив браузера матчей; новый
  мир — рестарт процесса (пустой стор → `newGame()`).

**Блок NET-P0 закрыт целиком** (порог победы 1100 в сети, совет учёных у всех мест,
боевые эффекты видны по сети, сессионный чат через сервер — session/коалиция/DM с
реплеем бэклога на джойне, переговоры между людьми консент-офферами: война →
предложение мира «⏳» → у второй стороны кнопка «✓ принять» → мир; проверено e2e
двумя живыми игроками).

**Не оставляй неаутентифицированный сервер публично надолго** (см.
`docs/multiplayer.md`, `render.yaml`) — плейтесты по ссылке-приглашению ок,
постоянный публичный сервер — после SE-1.x-клиента.
