# Баг-охота 2026-07-10 — триаж перед плейтестом

8 параллельных агентов-охотников по зонам; каждая находка подтверждалась временным
тестом против реального ядра (не только чтением кода). Ниже — сводный список по
серьёзности. Статус: ✅ исправлено · 🔧 в работе · ⏳ ждёт фикса.

Живая QA (CDP-прогон реального клиента в headless-браузере) дала **ноль
непойманных исключений в обоих локалях** — все находки ниже структурные/логические,
не краши.

## CRITICAL

- **BF-1** ✅ (PR #172) `[srv]` Гонка серверных драйверов: на durable-комнате сырой
  sync `submitAction` драйвера (ИИ/стоячие/Хранитель) мутировал `stateValue`/`seq`
  посреди `commitApply`-persist-await → применённое действие затиралось, seq
  откатывался. Фикс: `MatchRoom.submitServerAction` через mailbox на durable-пути.
- **BF-2** ⏳ `[cli]` `[srv]` Один `E_RATE_LIMIT` заклинивает gated-сессию НАВСЕГДА:
  `MultiplayerClient.sendAction` жжёт `clientSeq += 1` на каждый send и не откатывает
  при throttle/drop → строгий курсор гейта отвергает всё дальнейшее `E_OUT_OF_ORDER`.
  Триггер в плейтесте: мультиселект-групповой приказ >20 флотов трипает 20/1000ms.
  Фикс: клиент откатывает `clientSeq` на `E_RATE_LIMIT`/drop; и/или resync на
  `E_OUT_OF_ORDER`. Родственное: flood-guard транспорта ДРОПАЕТ >50/s без реджекта —
  та же потеря seq.
- **BF-3** ⏳ `[proto]` `fleet.merge` осиротляет `hero.fleetId` → бесплатное
  безлимитное дублирование геройского флагмана (`hero.spawn` не блокируется на stale
  fleetId + нет кулдауна respawn). Родственно — `fleet.split` может забрать hero-стек
  без репойнта. Фикс: репойнтить `hero.fleetId` в merge/split как репойнтятся дивизии;
  и/или `hero.spawn` отвергать при `alive === true`.
- **BF-4** ⏳ `[proto]` `[core]` `fleet.split` дублирует HP-пул повреждённого стека
  (`takeFromStacks` копирует весь `st.hp` на взятый стек, source не уменьшает), а
  `applyDamage` пересчитывает `count = ceil(pool/perShip)` без клампа → бой **плодит
  корабли**. NET-эксплойт-дупликатор. Фикс: делить пул пропорционально + клампить
  `newCount ≤ count`.
- **BF-5** ⏳ `[proto]` `mergeStacks`/`takeFromStacks` коалесцируют стеки по `(unit,hp)`
  игнорируя `modules`/`shieldHp` → оплаченный лоадаут размазывается на голые корпуса
  (минтинг статов) или уничтожается; split срезает модули с взятых. Ядровые
  `addUnits`/`findHealthyStack` делают правильно (по `loadoutKey`) — чинить прото-хелперы
  тем же ключом.

## MAJOR

### Бой (core)
- **BF-6** ⏳ `fleet.engage` (прото) мимо дипломатии: атака игрока в МИРЕ/пакте/альянсе
  (проверяется только `f.owner===target.owner`). Грифинг в мультиплеере.
- **BF-7** ⏳ Двойной наземный штурм одного гарнизона: нет «ground battle already live»
  гварда + `findEnemyFleetAt` пропускает battleId-флоты → два штурма делят defender-ref
  → дружеский огонь + двойной `planet.captured`.
- **BF-8** ⏳ Штурм при неразрешённой орбитальной битве (E_ORBIT_CONTESTED побеждён тем
  же battleId-слепым `findEnemyFleetAt`).
- **BF-9** ⏳ Нет chain-engage после наземной битвы: враждебные флоты вечно
  сосуществуют на узле (victor-блок только для `phase==='orbital'`).
- **BF-10** ⏳ Бомбардир, прикованный к мелэ (`battleId`), продолжает шелл планету, а
  ПВО его не видит (`nearOrbitHostile` пропускает battleId) — скрамбл защитника
  ЗАЩИЩАЕТ бомбардира.

### Ядро / время / детерминизм
- **BF-11** ⏳ `applyAction` перепрыгивает due-события (стампит `time` без их запуска),
  следующий `advanceTo` крутит часы НАЗАД и переначисляет прожитый спан (инфляция при
  залипшем catch-up). Фикс: reject `E_TIME_GAP` при `scheduled[0].at < now` или
  clamp `stepTime ≥ committed.time`; хосты — surface `partial`.
- **BF-12** ⏳ `schedule(NaN/Infinity)` принимается → бессмертное zombie-событие
  (не dead-letter), ломает `(at,seq)` и JSON-сериализуемость. Фикс: `schedule` —
  `reject('E_BAD_SCHEDULE')` при `!Number.isFinite(at)`.
- **BF-13** ⏳ Экономика (и hero/steward/intercept) итерируют `Object.keys` без сорта;
  Postgres JSONB-гибернация пересортировывает ключи → float-сложение в другом порядке
  → реплей/десинк-дивергенция. Фикс: канонизировать ключи на границе стора ИЛИ
  сортировать ключи в модулях (как victory/technology/artillery уже делают).
- **BF-14** ⏳ (MINOR→важно) `AdvanceResult.failures` (dead-letter) не читает ни один
  прод-код — коррупция без следа. Фикс: пробросить в `observe`/лог.

### Сеть / туман
- **BF-15** ⏳ `eventVisibleTo` режет целые классы событий ВСЕМ, включая владельца
  (`diplomacy.offered/changed`, `technology.researched`, `steward.*`, `market.*`,
  `player.eliminated` — payload-ключи не совпадают с `owner/location/planetId/at/fleetId`).
  Тосты «изучено»/утренний отчёт мертвы в NET. Фикс: матчить `playerId` + парные
  `a/b/from/to/buyer/seller`.
- **BF-16** ⏳ Приватный герой течёт сквозь туман: `hero.spawned/respawned` несут `at`
  (узел), `eventVisibleTo` раскрывает событие на опознанном узле → враг получает
  `{owner,heroId,fleetId,at}` для героя, которого проекция прячет. Фикс: `hero.*`
  события owner-only.
- **BF-17** ⏳ ИИ-замена без грейс-периода захватывает место человека при рестарте/дропе
  (`humans` пуст после рестарта → первый heartbeat гоняет expand-AI за место другого).
  Фикс: грейс-окно N минут, seed из claimed-мест стора.
- **BF-18** ⏳ Радар-сигнатуры не рендерятся в NET: сервер шлёт `signatures[]`, но
  `MultiplayerClient` их не парсит, а прототип считает блипы из `s.fleets` (где
  радар-онли врагов уже нет). Враги «выскакивают» из невидимости в опознанные. Фикс:
  прокинуть `signatures`/`remembered` через `MultiplayerSnapshot`.

### Наземка / герои / H4
- **BF-19** ⏳ Замок офицеров H4 — UI-only: `division.officer` живой и в gate-схеме →
  сырое действие цепляет любого офицера на дешёвую дивизию бесплатно / снимает с
  «запертого» премейда. Фикс: удалить хендлер+схему+билдер или отвергать не-премейд.
- **BF-20** ⏳ `division.rename` отсутствует в payloadSchemas → на gated-сервере
  (релиз-постура) переименование мёртво. `gateparity.test.ts` не обновлён H4. Фикс:
  добавить схему + оба parity-сэмпла (rename + officer-mobilize).
- **BF-21** ⏳ Офицерские премейды мобилизуются БЕЗ HP-бонуса офицера (`makeSide`
  без 3-го арг), но `regenDivision` его учитывает → рождается ниже своего же regen-max.
- **BF-22** ⏳ Наземный бой зависит от шага времени: daily-regen применяется раз-в-спан
  посреди боя → coarse vs fine stepping расходятся (ломает реплей/JSONB-паритет).
  Фикс: квантовать regen по тику ИЛИ гасить regen пока идёт `groundBattles[planetId]`.
- **BF-23** ⏳ Доктрины/синергии дизайнера дают ноль боевого эффекта (`formationStats`
  кормит только UI-превью; `groundTickAt` резолвит из `GROUND_ROSTER`+офицер).
  Фикс: либо завести `atkMul/defMul` в `mergeSide`, либо не рекламировать как бонусы.
- **BF-24** ⏳ Резервные (не задеплоенные) герои — неуязвимые кастеры: `hero.ability`
  их принимает (scan/аура/annihilate без риска и вне HERO_ACTIVE_CAP). Фикс: гейтить
  `hero.ability`/пассивы/reveal на `alive === true`.

### Флот / эскадрильи
- **BF-25** ⏳ Коллизия fleet-id молча удаляет корабли: `seq = keys.length`, delete
  освобождает номер → split→merge→split в тот же мс регенерирует существующий id и
  перезаписывает флот. Фикс: монотонный `fleetSeq` в state (как `battleSeq`).
- **BF-26** ⏳ `order.scramble` OFF→ON мгновенно заправляет сухое/перезаряжающееся
  крыло (`freshSortie(max)` всегда), обходя весь sortie/rearm-механизм. Wire-legal →
  NET пропускает кулдаун бесплатно. Фикс: хранить sortie на флоте, не только в patrol.
- **BF-27** ⏳ Гарнизон под наземным штурмом эвакуируется (`army.load`/`fleet.launch`
  без battle-lock) → атакующий выигрывает без потерь, защитник уходит от резолва.
  Фикс: отвергать load/launch пока жива битва с `ref{garrison,planetId}`.
- **BF-28** ⏳ `fleet.launch` полностью обходит cargo-cap (поднял 20 танков за один
  крузер как `landing`). Фикс: капить `landing` по `Σ cargoCapacity`.
- **BF-29** ⏳ Авто-rally срезает оплаченный лоадаут с построенного корабля
  (`unit.built` несёт `modules`, rally-хендлер их теряет) — «Оснащение» = ресурс-сток.
- **BF-30** ⏳ Патруль жжёт топливо по цели, уже запертой в бою (`fleet.engage` →
  `E_IN_BATTLE`, но `patch{fuel-1}` персистится до/вне исхода). Враг может запереть
  жертвенный флот в радиусе патруля и осушить его навсегда. Фикс: фильтровать
  `g.battleId` из целей (оба драйвера).

### Дипломатия / победа
- **BF-31** ✅ (PR #172) Коалиция = связная компонента (транзитивная) вместо клики:
  цепочка A–B–C при войне A–C побеждала вместе; непричастный третий блокировал победу.
  Фикс: коалиция — клика (все пары взаимно alliance).
- **BF-32** ⏳ `diplomacy.declare` не проверяет `status==='active'` актора/цели:
  побеждённые остаются полными дип-акторами, офферы к мёртвым висят вечно (sweep
  только в момент elimination). Фикс: гейт статуса как в espionage.
- **BF-33** ⏳ Бот-дипломатия: гвард мёртвого места (`if (!players[bot])`) — dead-code
  (elimination ставит `defeated`, не удаляет запись) → побеждённый бот шлёт войну «из
  могилы» и отвечает на офферы. Фикс: `players[bot]?.status !== 'active'` + sweep
  `approval` на elimination.
- **BF-34** ⏳ (MINOR) Нет терминального гейта после конца матча: kernel/gate/room
  применяют действия при `match.status==='ended'` (мир мутирует и персистится
  бесконечно). Нужен дизайн-колл про grace-период.
- **BF-35** ⏳ (MINOR) `netIncome` HUD не учитывает `economy.production`-хуки
  faction/tech → у всех мест (фракция blue +12%) верхняя `+/ч` неверна с 1-й минуты.
