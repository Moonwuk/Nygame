# OWASP A06:2025 — Insecure Design: модель угроз + план закрытия

> A06 — единственная категория Top-10, которую **сканеры не ловят** (`security.yml`:
> Semgrep/CodeQL/Trivy/… ищут баги реализации, не дизайна). Закрывается **процессом**
> (модель угроз + misuse-кейсы на фичу) и **дизайн-контролами**, а не патчами. Этот док —
> живая модель угроз: активы, границы доверия, misuse-кейсы → контролы, и брики починки.
> Парные доки: `architecture.md` (§безопасность), `multiplayer.md`, `deep-technical-roadmap.md`,
> `accounts-roadmap.md`, `CLAUDE.md` (инварианты). Формат брика: `id · зона · статус · «Готово, когда»`.

---

## 0. Что уже есть (базис secure-by-design)

Костяк A06 у нас реально заложен инвариантами, не на словах:

- **Server-authority** — клиент шлёт намерение, не состояние (`CLAUDE.md` инв. 5). Фундамент анти-чита.
- **Fail-secure** — любая ошибка → `{ok:false, code}`, без частичной мутации (инв. 4). **Откат не нужен by design**: редьюсер чист, работает на `deepClone`-черновике; apply-or-reject атомарен.
- **Туман = граница безопасности** — `visibleState` (`state/visibility.ts`) физически вырезает чужое на сервере (казна/тех/учёный/скор, скрытые планеты, невидимые флоты/бои, **всё расписание**); радар → грубые бакеты. События ре-фильтруются per-player (`matchRoom.ts`).
- **Валидация на границе** — весь внешний вход через zod / `safeParseGameData` до ядра (A05/A08).
- **Детерминизм = контроль целостности** — `hashState` ловит десинк/подмену; реплей воспроизводим.
- **Изоляция по матчу** — каждый `MatchRoom` самодостаточен (свой kernel + GameState-JSONB); общего мутабельного мира нет.
- **DoS-границы (F-03/F-04)** — `matchRoom.ts`: rate-limit 20/1000мс, 10k FIFO-квитанций, 32KB-кап payload, backpressure, ping-капы; `action.playerId == connection`.
- **Экономика сохраняется** — рынок эскроует, self-trade отбит, **минта нет**; военная сила очков не даёт (победу не «купить»).

## 1. Границы доверия и активы

| Граница | Кто по разные стороны | Актив за ней |
|---|---|---|
| клиент ↔ сервер | недоверенный клиент ↔ авторитетное ядро | правила боя/экономики, чужое состояние |
| игрок ↔ игрок (в матче) | соперники | туман (позиции/состав/казна/планы) |
| матч ↔ матч | независимые комнаты | изоляция (ресурсы/скор одного матча не текут в другой) |
| аккаунт ↔ аккаунт | **auth построен (JWT, за `AUTH_JWT_SECRET`); анти-альт — открыт** | идентичность, анти-альт, премиум-валюта |
| матч ↔ мета | комната ↔ глобальный слой (north-star) | глобальный рынок/лидерборд |

## 2. Брики починки (по приоритету)

### ✅ SEC-A06-1 · Fog-aware reject-коды (side-channel перечисления) `[core]` — сделано
**Угроза:** id последовательны; разница `E_NO_FLEET` vs `E_FORBIDDEN` (own-fleet действия) и
`E_NO_TARGET` vs `E_NOT_HOSTILE` (цель barrage) позволяла перебором подтверждать **существование
и военную стойку флотов, скрытых туманом**.
**Сделано:** «нет флота» и «не твой флот» отвечают **одним** `E_NO_FLEET`; несуществующая и
не-враждебная цель — одним `E_NO_TARGET`. Схлопнуто в `requireOwnedIdleFleet` (orbit/assault/bombard),
`fleet.move`/`stop`, `fleet.barrage`/`barrageMode`/`retreat`, цель barrage. `E_FLEET_BUSY` достижим
только для своего флота → не течёт. Тесты обновлены на непрозрачные коды.

### ✅ SEC-A06-2 · Кап открытых ордеров на игрока `[core]` — сделано
**Угроза:** публичный, реплицируемый всем клиентам стакан; сток, раздроблённый на копеечные
листинги, пухнет в состоянии и в каждом payload (только rate-limit ограничивал).
**Сделано:** `MAX_OPEN_ORDERS = 20` на продавца в `market.list` → `E_ORDER_LIMIT`. +тест.

### ⏳ SEC-A06-3 · Систематический аудит side-channel по ВСЕМ entity-id `[core]` — S/M
**Угроза:** тот же приём перечисления применим к **другим** id, а не только флотам — герои
(`hero.*` отдаёт `E_FORBIDDEN` на чужого героя, а герои fog-стрипаются), возможно ордера/планеты.
**Решение:** пройти каждое действие, что резолвит сущность по клиентскому id, и для fog-gated
сущностей сделать «не найдено == не твоё == не видно» одним кодом. Ввести общий хелпер/паттерн
(`denyOpaque`) и, по возможности, ESLint-правило против «`E_FORBIDDEN` после lookup по client id».
Тайминг-side-channel (различие по времени) — отметить как низкий приоритет (редьюсер грубозернист).
**Готово, когда:** ни одно действие не различает существование fog-скрытой сущности по коду; тест-матрица.

### ⏳ SEC-A06-4 · Дипломатия: агро-огонь по миру + объявление войны `[core]` — M
**Угроза:** `aggressive`-артиллерия бьёт партнёра по **миру** без ответного огня (`artillery.ts` runArtillery),
а ближний бой требует **войны**. Объявление войны **сделано**: `diplomacy.declare {target, stance}`
(`diplomacy.ts:58`) — эскалация к войне односторонняя, к миру/пакту/союзу — по обоюдному согласию;
жертва может ответить. **Остаётся:** гейтить `aggressive`-артиллерию за войну (сейчас бьёт партнёра
по миру без ответного огня — `artillery.ts` runArtillery); misuse-тест «shell-a-peace-partner».
**Готово, когда:** нельзя безнаказанно бить не-врага; жертва может объявить войну; misuse-тест «shell-a-peace-partner».

### ⏳ SEC-A06-5 · Рынок: цена ≥ 1 / анти-wash-trade `[core]` (гейт на identity) — S/L
**Угроза:** `price=0` разрешён и стойка не проверяется (`market.ts:43`) → с появлением **альтов**
это отмыв/перелив ресурсов между своими аккаунтами всего за 15% burn.
**Решение:** дешёвый guard сейчас — `price ≥ 1` (запрет буквально-бесплатных листингов); полный фикс
завязан на identity (SEC-A06-7): лимиты трансфера на аккаунт, стойка-гейт (не дарить союзнику даром),
эвристики аномалий. **Готово, когда:** бесплатных листингов нет; после identity — лимиты/детект в дизайне.

### ⏳ SEC-A06-6 · Инфляция знаменателя победы (`station.deploy`) `[core]` — S
**Угроза:** `station.deploy` плодит capturable `void_station` (`station.ts`, `sectorKinds.json`),
раздувая знаменатель доминирования (`victory.ts:261-267` считает только capturable) → проигрывающий
может **надуть 60%-бар оппонента** и оттянуть/сорвать победу.
**Решение:** исключить искусственные `void_station` (или не-исходно-владетые структуры) из знаменателя
доминирования, либо считать только «природно-capturable» секторы. Свериться с логикой `dead_world`.
**Готово, когда:** депой станций не двигает чужой бар доминирования; тест.

### ⏳ SEC-A06-7 · Идентичность, сессия, анти-альт (крупнейшая зона A06) `[srv][design]` — L
**Угроза:** auth/JWT **построен, но опционален** (за `AUTH_JWT_SECRET`, `auth.ts`): при включении
`playerId` криптографически связан join-токеном на хендшейке (`wsServer.ts:243` verifyJoinToken), в
dev-режиме без секрета — держится на биндинге к соединению (`matchRoom.ts`). Остаётся: угон сессии;
**мульти-аккаунт / Sybil / фидинг** (один человек гоняет N аккаунтов, сливая ресурсы/территорию в
основной — rate-limit это НЕ ловит); фрод премиум-валюты.
**Решение:** (а) ✅ базово: `action-layer` envelope-гейт (`GATE=1`) и JWT-auth (`AUTH_JWT_SECRET`)
уже подключены — `playerId` связан токеном; остаётся сильная identity (OIDC, `accounts-roadmap.md`);
(б) **дизайн против альт-абьюза** —
misuse-док, лимиты на аккаунт, правила рынка/трейда (см. SEC-A06-5), эвристики детекта, либо явное
решение «принимаем». **Готово, когда:** identity доказуема; есть осознанная политика по альтам.

### ⏳ SEC-A06-8 · Soft-капы роста состояния `[core]` — S/M
**Угроза:** `unit.build` берёт любой положительный count → огромный конечный стек за дёшево;
цикл move/stop плодит устаревшие `fleet.intercept` в `scheduled` (само-проверка их no-op-ит, но они
копятся). Ограничено только rate-limit, жёсткого капа нет.
**Решение:** (а) кап count на заказ постройки (или доверять cost-гейту + разумный максимум);
(б) дедуп/прунинг intercept-событий — один pending на пару флот-лейн, чистить устаревшие при переприказе.
**Готово, когда:** ни одно действие не растит `scheduled`/стеки без границы; тест.

### 🔒 SEC-A06-9 · Кросс-матчевые фичи = новая граница доверия `[design]` — north-star
**Угроза:** глобальный рынок/лидерборд/мета-галактика ломают чистую изоляцию-по-матчу; общий сток →
альт-абьюз/wash-trade межматчево.
**Решение:** при постройке — своя модель угроз на каждый общий слой; кросс-матчевый переток ресурсов
трактовать как поверхность альт-абьюза «в большом». **Готово, когда:** каждая мета-фича имеет свой threat-model.

### ⏳ SEC-A06-10 · Процесс: misuse-гейт + живой threat-model `[process]` — S
**Угроза (мета):** безопасность сильная, но **неявная** (размазана по инвариантам). A06 требует
артефакта процесса.
**Решение:** (а) на каждый кирпич — чек «как это заабьюзит читер/грифер?» рядом с тестами (в духе
`behavioral-guidelines` из `CLAUDE.md`); (б) держать этот док живым (активы/границы/misuse → контролы);
(в) при добавлении действия — обязательный пункт «reject-коды не текут» (SEC-A06-1/3).
**Готово, когда:** ни одна новая механика не мержится без misuse-заметки; док обновляется вместе с кодом.

## 3. Осознанно принятые компромиссы (не баги)

- **Публичный стакан рынка** раскрывает существование fogged-игрока в момент листинга — это природа
  **биржи** (публична by design). Принято; не «утечка», а свойство механики.
- **Отступление ниже 40% корпуса убивает флот** — не A06, а балансный дизайн («не тяни до последнего»).

## 4. Последовательность

`SEC-A06-1/2` (сделано) → `SEC-A06-10` (процесс — включить сразу, дёшево) → `SEC-A06-3` (закрыть
side-channel до конца) → `SEC-A06-4`/`6`/`8` (точечные дизайн-дыры) → `SEC-A06-7` (identity + анти-альт,
крупная, гейтит `SEC-A06-5`) → `SEC-A06-9` (мета, по мере постройки кросс-матчевых фич).
