# Войны корпораций (AvA) — сессионная битва · MVP

> Детальный жизненный цикл **AvA** (alliance-vs-alliance) — расшивка Контура 2 из
> `metagame.md`. **Первый срез: чистая СЕССИОННАЯ битва корпорация-vs-корпорация,
> БЕЗ ставки мета-сектора.** Территориальный приз (мета-галактика, владение сектором,
> пассивные бонусы) — будущий слой, не в этом MVP. Слой — **server/meta**;
> детерминированное ядро не трогаем (на выходе оркестратора — обычный матч-конфиг).
> Парные доки: `metagame.md` (Контур 2), `matchmaking-roadmap.md` (жизненный цикл
> матча), `map-roadmap.md` (карта/слоты), `gdd.md` (§2 консервация, §3 timeScale,
> §5.2 снапшот прогресса), `main-menu.md` (вход в AvA из мета-шелла).

---

## 1. Что это — и чего пока НЕТ

AvA — организованная битва двух **готовых** корпораций на инстансной карте.

**Граница MVP (важно):**

- ✗ НЕТ мета-галактики, территориальной ставки и привязки к конкретному мета-сектору
  (`metagame.md` Контур 2 в полном виде) — это «потом».
- ✓ Победа даёт **обычный пост-матч**: опыт аккаунта, очки влияния в корпорацию,
  движение в рейтинге корпораций (`main-menu.md` §4.6). Захвата мета-сектора пока нет.

## 2. Два флага готовности

| Флаг | Кто ставит | Что даёт |
|---|---|---|
| **Корпоративный** | глава корпорации | корпа попадает в **пул «готовых к AvA»** (видна для вызова / может вызывать). Снимается главой. |
| **Игровой** | игрок (внутри своей корпы) | игрок в **пуле развёртывания** корпы. Флаг = **постоянное согласие**: офицеры могут включить его в состав и развернуть **без его активного участия** (даже офлайн). |

**Без игрового флага** игрок попадает в бой только если **сам** зайдёт / поставит
готовность во время сбора. Флаг = согласие; нет флага — не развернуть.

## 3. Жизненный цикл

```
(S0) Пул готовых        корпа с корп-флагом ждёт/ищет соперника
        │
(S1) Вызов              готовая корпа вызывает другую готовую; стоит ОЧКИ ВЛИЯНИЯ корпы
        │                    └─ цель: ПРИНЯТЬ / ОТКЛОНИТЬ (вызов истекает по таймеру)
        ▼
(S2) Законтрактовано    обе стороны согласны
        │
(S3) ПАУЗА (сбор) 1–2 усл. дня
        │   набор состава ОБЕИХ сторон:
        │     • флагнутых [§2] офицеры включают в ростер → авто-развёртывание
        │     • нефлагнутые — только самозапись (зашёл / поставил готовность)
        │   в конце паузы РОСТЕР ЛОЧИТСЯ (GDD §2 консервация)
        ▼
(S4) Сессия + карта     карта РАНДОМОМ из пула AvA-карт; игроки рассажены в
        │               командные слоты; спавн по политике карты (§4)
        ▼
(S5) МИРНЫЙ ПЕРИОД      сессия живая, БОЙ ЗАЛОЧЕН: экономика / стройка / движение /
        │               укрепление идут; timeScale-часы тикают
        ▼
(S6) ВОЙНА              бой открывается; идёт до победы стороны / триггеров финала (GDD §3.2)
        ▼
(S7) Итог               пост-матч награды (§1); сессия архивируется (MM-3.1)
```

> **Граница фаз (подтверждено):** **пауза** = до-сессионный сбор ростера (сессии/карты
> ещё нет, в конце состав лочится); **мирный период** = уже живая сессия с залоченным
> боем (подготовка экономики). Порядок: **сбор → сессия → мир**.

## 4. Карта, команды и спавн

- **Пул AvA-карт:** карты `data/maps/*`, помеченные как AvA-eligible (тег + размер
  под число слотов). Оркестратор выбирает одну случайно на S4.
- **⚠️ Пробел в текущей схеме.** `mapSchema.ts` сейчас **вшивает конкретных игроков**
  (`players`) и их `owner`-сектора — стартовые позиции запечены в карту. Для AvA нужна
  абстракция **слотов**, отвязанная от конкретных id:
  - карта объявляет **N стартовых слотов** с **командной принадлежностью** (сторона A / B);
  - у слота/карты — **политика спавна**: `fixed` / `choice` / `random`
    («спавн случайный или по выбору — зависит от карты»);
  - оркестратор **сажает живых игроков в слоты** при создании сессии (S4).
  - **Команд в схеме сейчас нет** — это новый кусок (слой слотов/команд над картой).
- Союзники одной стороны должны спавниться **сгруппированно** (единый фронт) —
  свойство раскладки слотов карты.

## 5. Баланс сторон (MVP-дефолты · открыто)

- Стороны **симметричны по числу слотов** (задаёт карта); ростер капается слотами.
- **Минимум на сторону** для старта; недобор по концу паузы → отмена матча
  (вызов/цена возвращается).
- Переполнение (желающих больше, чем слотов) → не сажаем сверх слотов (резерв/очередь).
- Хендикап и рейтинговый подбор корпораций по силе — **отложено** (не в MVP).

## 6. Пауза и мирный период — семантика (MVP-дефолты)

- Длительности — в **timeScale-днях** (как все таймеры матча, GDD §3.1). Пауза = 1–2
  условных дня; длительность мирного периода — TBD (§10).
- **Пауза (сбор):** ростер филлится; в конце **лочится** — после этого состав/дека не
  меняются (GDD §2 консервация: фиксация при старте, смена → отклоняется сервером).
- **Мирный период:** **combat-lock**; разрешены экономика / стройка / движение /
  оборона; мировые часы идут (накопление по `time.advanced`). Затем бой открывается.
- **Поздний вход:** только в окне **паузы**; после лока — нет. Реинфорс/замены после
  начала войны — **отложено**.

## 7. Офлайн авто-развёртывание (MVP-дефолты)

Развёрнутый, но отсутствующий игрок не должен быть ни мёртвым грузом, ни авто-агрессором:

- **Пассивная экономика** идёт по движку (`economy` на `time.advanced`); **гарнизон
  обороняется** (бой разрешается детерминированно без игрока, `resolveBattle`).
- **Никакого авто-микро/AI-наступления** — атакующие действия только от вошедшего игрока.
- **Лоадаут = снапшот личного арсенала** на момент развёртывания (GDD §5.2,
  snapshot-at-start) + опц. аренда корп-снаряжения (`metagame.md`, когда появится).

## 8. Где это живёт (архитектура)

- **server/meta:** пул готовых, флаги, вызовы/принятие, ростеры, оркестрация фаз
  (S0–S7), выдача наград. Авторитетно (server-authority); все интенты **fail-secure**
  — отказ со стабильным кодом, без утечки причин (A10).
- **Переиспользует** жизненный цикл матча (`matchmaking-roadmap.md` `MM-0.1`:
  lobby→active→ended→archived). AvA — **свой режим матчмейкинга** рядом с обычным
  подбором (`MM-2.1`); пауза/мир — под-фазы старта матча. Итог пишется в историю
  аккаунтов (`MM-3.1`).
- **Ядро (`shared-core`):** ничего про AvA не знает — получает матч-конфиг (карта +
  команды + рассадка), отдаёт результат. Детерминизм не затронут.
- **Публичная лента новостей:** оркестратор публикует в общую ленту корпораций
  (`main-menu.md` §4.7) **матчап на S2** (бой подтверждён, обе корпы готовы — не
  pending-вызов S1) и **итог на S7** (кто победил). Только публичные факты (без
  приватного ростера), read-model, server-authority.

## 9. Интенты (клиент → сервер)

```
setCorpReady / clearCorpReady          (глава)         — корп-флаг, пул готовых
setPlayerReady / clearPlayerReady      (игрок)         — игровой флаг, пул развёртывания
challengeCorp(targetCorpId)            (глава)         — вызов
acceptChallenge / declineChallenge     (глава цели)    — реакция на вызов
setRoster(playerIds[])                 (глава/офицер)  — состав из флагнутых
joinAva / readyUp                      (игрок)         — самозапись в окне паузы
```

Развёртывание на S4 — **автоматическое** (из ростера), отдельного интента игрока не требует.
