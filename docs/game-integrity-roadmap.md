# Целостность игры (анти-чит и честность) — технический roadmap

> **Блок 6/7** из `tech-research.md` (§8 анти-чит). Честность авторитетного **async**-мира:
> композиция уже заложенных контролей (server-authority, идемпотентность, туман) + игровая
> детекция (rate-limit, аномалии, мультиаккаунт/сговор, боты, эксплойты экономики, аудит-реплей).
> Рантайм-механика контролей — в `secure-environment-roadmap.md` (SE-6); этот блок — **игровая**
> сторона (что считать читом, как ловить, как реагировать). Формат — кирпичики. Соразмерно жанру
> (нет twitch-аима). Статусы: ✅ · ⏳ · 🔒(dep).

## Текущее состояние (факт)

Заложен фундамент честности: **server-authority** (клиент шлёт намерение, не состояние),
детерминизм, идемпотентность/anti-replay (action-layer), `visibleState` (анти-maphack в ядре),
**туман-на-отправке** (per-player `visibleState` + дельта считаются на сервере — невидимые
миры/флоты физически не уходят клиенту; ✅ уже в коде: `packages/server/src/matchRoom.ts`
`viewFor`/`broadcastState`), базовый per-player rate-limit (`E_RATE_LIMIT`,
`packages/server/src/matchRoom.ts`), fail-secure. **Нет:** anti-cheat rate-shaping (детекции
аномалий), анти-мультиаккаунта/сговора, bot-detection, per-player очереди (double-spend).

## Карта угроз → контроль (STRIDE/игровое)

| Угроза | Контроль | Где |
| --- | --- | --- |
| Подделка интентов (S/T) | server-authority + per-message authz | ✅ инвариант / SV-1.1 |
| Double-spend / TOCTOU (T) | per-player очередь (concurrency=1) | 🔒 F5/SE-6.3 |
| Maphack / утечка тумана (I) | `visibleState` **на отправке** | ✅ (server `viewFor`/`broadcastState`; остаётся byte-level anti-leak тест) |
| Replay / abuse идемпотентности | sequence-gate + receipts | ✅ (in-mem) |
| Флуд/спам/бот (DoS экономики) | rate-limit + аномалии + bot-detect | 🔴 этот блок |
| Мультиаккаунт/сговор | детекция связей + правила | 🔴 этот блок |

## Фаза 0 · Базовые контроли честности (композиция)

### GI-0.1 · Rate-limiting действий `[srv][sec]` 🔒(F4) — M
**Цель:** анти-спам/бот. **(Игровая настройка поверх SE-6.2.)**
**Подзадачи:** лимиты per-account/per-connection с учётом жанра (пик ~3 действия/сек — щедро); разные классы действий; превышение → отказ + лог-аномалия (SE-8).
**Готово, когда:** флуд действий ограничивается и логируется.

### GI-0.2 · Per-player очередь (анти-double-spend) `[srv]` 🔒(F2) — M
**Подзадачи:** сериализация действий игрока (= SV-2.3 / F5 / SE-6.3); атомарность экономических операций (нет check-then-act).
**Готово, когда:** двойная трата невозможна; тест гонки экономики.

### GI-0.3 · Туман как граница (анти-maphack) `[srv]` ✅ (в основном) — M
**✅ уже в коде:** `packages/server/src/matchRoom.ts` — `viewFor` считает per-player `visibleState`
на сервере, `broadcastState` шлёт каждому игроку только дельту против его собственного видимого
вида (невидимые миры/флоты физически не отправляются); события тоже фильтруются туманом
(`eventVisibleTo`). Не полагается на клиент.
**Подзадачи:** ~~`visibleState` на отправке (= SV-3.1 / F6)~~ ✅; ~~не полагаться на клиент~~ ✅;
остаётся тест anti-leak по байтам.
**Готово, когда:** невидимые данные физически не уходят клиенту.

## Фаза 1 · Детекция аномалий

### GI-1.1 · Сигналы и метрики честности `[srv][sec]` 🔒(SE-8.1) — M
**Цель:** видеть подозрительное.
**Подзадачи:** счётчики отклонённых действий, превышений rate-limit, ошибок авторизации на игрока/матч; экспорт в метрики/логи (SE-8).
**Готово, когда:** по игроку/матчу видны сигналы честности.

### GI-1.2 · Алерты на аномалии `[sec]` 🔒(GI-1.1) — M
**Подзадачи:** алерт на всплеск отклонённых/ошибок авторизации (= чит/атака, A09); пороги; маршрутизация. **(= SE-8.2.)**
**Готово, когда:** аномальный паттерн поднимает алерт.

### GI-1.3 · Аудит-реплей подозрительного матча `[srv]` 🔒(PE-1.1) — M
**Цель:** разобрать инцидент детерминированно.
**Подзадачи:** из event-log (PE-1.1) пересобрать ход матча; инструмент «что делал игрок X»; пометка/откат при подтверждённом эксплойте.
**Готово, когда:** по реплею воспроизводится спорная ситуация; есть процедура реакции.

## Фаза 2 · Мультиаккаунт, сговор, боты

### GI-2.1 · Анти-мультиаккаунт / связи `[srv][sec]` 🔒(AC-0.1) — L
**Цель:** один игрок ≠ несколько мест в одном матче / фарм-аккаунты.
**Подзадачи:** сигналы связей (устройство/сеть/поведение — без избыточного PII, SE-10); правила в одном матче; rate-limit на создание аккаунтов (force adversary cost).
**Готово, когда:** очевидные мультиаккаунты в матче ловятся/блокируются.

### GI-2.2 · Поведенческая bot-detection `[srv]` 🔒(GI-1.1) — L
**Цель:** ловить автоматизацию/фарм соразмерно жанру.
**Подзадачи:** поведенческие признаки (регулярность/скорость/паттерны действий); скоринг; ручной разбор по реплею; постепенный энфорсмент (анти-ложноположительные).
**Готово, когда:** явные боты получают скоринг и разбор.

### GI-2.3 · Защита экономики/обменов `[srv][core]` 🔒(GI-0.2) — M
**Цель:** анти-дюп/анти-эксплойт экономики и (будущих) обменов.
**Подзадачи:** транзакционная атомарность экономических действий; инварианты «нельзя создать из воздуха»; проверка при обменах/дипломатии; регресс-тесты на найденные эксплойты.
**Готово, когда:** экономические инварианты держатся под гонками/злоупотреблениями.
