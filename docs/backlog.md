# Бэклог-кирпичики — как делим работу

> Гранулярные, назначаемые задачи. **1 кирпичик ≈ 1 PR ≈ 1 сессия Claude**, в
> пределах **одной зоны** (пакет / модуль / данные) — чтобы параллельная работа не
> конфликтовала. Регламент — `CONTRIBUTING.md`; высокоуровневый план —
> `roadmap.md` и `deep-technical-roadmap.md`; что уже готово — `state.md`.

## Как взять кирпичик

1. Возьми кирпичик со статусом ⏳ (не 🔒 — у того ещё не закрыты зависимости).
2. Застолби его (issue/доска), помечен зоной — чтобы не пересечься с напарником.
3. Ветка от `main` → код + тесты → **`pnpm run check`** зелёный → PR в `main`.
4. **Один кирпичик — одна зона.** Не бери два кирпича в одном файле одновременно.
5. Закрыл — отметь ✅ здесь и обнови `state.md` (в том же PR).

## Зоны

`[core]` `packages/shared-core/src/modules` (+ `data`) · `[act]` `action-layer` ·
`[srv]` `server` · `[cli]` `client` · `[proto]` `prototype` · `[data]` `data/*.json` ·
`[docs]` `docs` · `[sec]` CI/сканеры (`.github/workflows/security.yml`, `docs/security`, конфиги сканеров) ·
`[ops]` эксплуатация/хостинг (запуск сервера, автообновление, инфраструктура плейтестов)

Словарь зон машинно проверяется гейтом (`scripts/docs-check.mjs`) — вместе с
целостностью всех `.md`-ссылок в docs/: новая зона сначала добавляется сюда и в
`ZONES` скрипта, иначе `pnpm run check` красный.

## Статусы

✅ готово · ⏳ можно брать · 🔒 ждёт зависимость (в скобках — какую)

> **Фундамент готов** (не переделывать): микроядро/шина/хуки/манифест, RNG+golden,
> модель времени, экономика, карта+движение (incl. `fleet.stop`), секторы, бой
> (орбита/десант/захват/бомбардировка/ПВО), здания, флот⊕армия+транспорт, **типы
> планет**, **victory-модуль** (data-driven очки), **`visibleState`** (туман войны),
> каркас **action-layer**, multiplayer-slice, **F6 туман на рассылке**, играбельный прототип,
> **дипломатия в ядре** (D1+D2+D3), **предматчевые технологии** (C3), радарные бонусы
> тех/фракций (A2), **шпионаж-фундамент** (SPY-1). Тесты зелёные (актуальный счётчик — в
> шапке `state.md`). Блоки A, B, C, D закрыты целиком — ядро (этап 1) собрано.

  активного исследования на `initialPercent × decay^boosts` (деф. 25%, затухание
  ×0.5 — геометрически убывающая эффективность, мгновенного завершения не купить;
  НЕ юниты/боевая мощь/опыт героев). Ускорение = решедул: `completesAt` сдвигается,
  старое `technology.complete` становится no-op по несовпавшему `completesAt`.
  Конфиг `GameData.researchBoost` (`ResearchBoostDefSchema`, zod-default — бандлы
  без блока не тронуты), payload-схема в гейте. Коды: `E_NOT_ACTIVE`, `E_TOO_LATE`,
  `E_INSUFFICIENT`, `E_BAD_PAYLOAD`. Осталось за скоупом: расстановка energy_nexus
  на картах (map-дизайн), прото-UI кнопки буста, источник 2 (покупка на мета-рынке
  — EC-*).
- **EFX-1** ✅ `[core]` **Универсальный движок трейтов/эффектов.** `effectsModule` —
  интерпретатор `data.events` (`EffectRuleSchema`): `planet_captured` — trait-scoped
  (правило = трейт: срабатывает, когда захватившая сила несёт id правила в `traits`
  юнита — ровно пример architecture.md §2.2), `schedule` — глобальное тёмное событие
  по абсолютной сетке `cadenceHours` (детерминированно через `time.advanced`-спаны,
  каждый активный игрок катает `chance` независимо; cap 100 срабатываний/спан).
  Эффекты — через capability `effect.<name>` (шов расширения) с фолбэком на встроенные
  `add_trait`/`modify_resource` (кламп ≥0); неизвестный trigger/effect → правило
  инертно, никогда не краш; каждое срабатывание эмитит `effect.applied`. Юнитовые
  `traits` теперь читаются генерически (`data/traits.ts`: `defHasTrait`/`unitHasTrait`/
  `stacksHaveTrait`; artillery/immobile/hero переведены на общий словарь). Подключён в
  кернел прототипа (инертен при `events: {}`). 11 тестов. Разгрёб главный дизайн-долг
  Этапа 1.

## Блок AVA · AvA-жизненный цикл (server/meta) `[srv]`

> Полный жизненный цикл Alliance-vs-Alliance из `corporation-wars.md` (мета-карта в
> долгий ящик). Слой — server/meta, вне детерминированного ядра. Лесенка: **база
> корпораций** → флаги готовности + вызов/принятие → ростер + оркестрация фаз.
> **Детальный технический роадмап (опоры/модель данных/API/тесты по каждому кирпичу) —
> `docs/ava-lifecycle-roadmap.md`.** Крит-путь: AVA-1 → AVA-3 → AVA-4 → AVA-6 → AVA-7 → AVA-8.

- **CORP-0** ✅ `[srv]` **База корпораций (членство/роли RBAC + store + REST).**
  Организационный слой из `corporations.md`: `CorpStore` (Memory + Postgres) —
  корпорации, членство `head|officer|member|recruit` (рекрут-строка = заявка),
  аудит; структурные инварианты на уровне хранилища (одна корпа на аккаунт — PK по
  `account_id`; уникальное имя без учёта регистра; атомарные `createCorp`/`swapHead`
  в транзакции; аудит переживает роспуск). `CorpService` применяет матрицу прав
  `corporations.md` §2 fail-secure стабильными кодами (`E_FORBIDDEN`/`E_NOT_MEMBER`/
  `E_HEAD_MUST_TRANSFER`/`E_NAME_TAKEN`/`E_IN_CORP`/…): ровно один Глава, Главу не
  кикнуть, офицер не эскалирует (не меняет роли, не трогает Главу/офицера), передача
  главенства — отдельное явное действие, Глава уходит только передав главенство (или в
  одиночку — это роспуск). REST в `main.ts` (session-gated на каждом роуте, per-IP
  rate-limit на записи по образцу auth/match API, аудит-лог). Контрактные тесты обоих
  адаптеров (memory + Postgres) + вся матрица прав + HTTP-контракт. **Отложено (не
  спекулятивно):** медали (нужна история матчей MM-3.1), гейт `createCorp` по уровню
  аккаунта (нужен серверный XP/уровень AC-0.3). **Дальше:** флаги готовности к AvA
  (корп-флаг Главы + player-consent) + вызов/принятие (тратит очки влияния, истечение);
  затем ростер (пауза/лок) + оркестрация фаз S3–S7 через существующий MatchRegistry +
  посев команд AVA-0.
- **AVA-1** ✅ `[srv/core]` **Командная дипломатия на серверном пути.** `buildStateFromMap`
  сеет `state.diplomacy` из `slot.team` (хелпер `seedTeamDiplomacy`, тот же посев, что
  прото-`newGame`): карта без команд → peace-FFA; та же сторона → `alliance` (seeded —

  `peace` — мирный старт AvA под AVA-8). Пары из отсортированных id — канонический JSON.
  Хвост **NET-2v2** тоже: `TEAMS=2v2` на прото-хосте сеет 4 командных seats (A: p1+p2
  западные углы, B: p3+p4 восточные) — живой 2v2 до оркестратора; пустые кресла играет
  серверный ИИ. Тесты в `buildFromMap.test.ts` (war/peace/2v2/FFA/детерминизм).
- **AVA-2** ✅ `[srv]` **Очки влияния корпорации.** Корп-валюта `influence` в `CorpStore`

- **AVA-3** ✅ `[srv]` **Флаги готовности к AvA.** Корп-флаг (глава → пул готовых,

- **AVA-4** ✅ `[srv]` **Вызов/принятие (S0–S2).** `AvaService.challenge` (глава готовой

- **AVA-5** ✅ `[core/data]` **Пул AvA-карт + eligibility.** `MatchMapSchema.avaEligible`

- **AVA-6** ✅ `[srv]` **Сбор ростера + лок (S3).** `accept` теперь ОТКРЫВАЕТ окно паузы

  `escalateWar` (main: `registry.resolve` будит комнату, `warDeclarationsFor(state)` —
  чистые системные декларации ровно по кросс-командным peace-парам с детерминированными

  функцию вместо инлайн-условий; баннер запуска печатает окно грейса. 3 теста
  (таблица истинности: Хранитель бьёт присутствие и грейс; присутствующий не
  вытесняется; пустое кресло ждёт грейс → заместитель). Живой смоук (grace=0 →
  пустые кресла играются; ×200 мир идёт).
- **SES-2.3** ✅ `[srv]` **Окно входа 4 реальных дня.** Новое кресло можно занять
  только в течение окна от создания сессии; занятое место (свой тикет/ник) —
  реконнект в любой момент. Транспорт: опция `admitNewSeat?(matchId)` в `wsServer` —
  вызывается ТОЛЬКО для первого входа ника (проверка через неассигнящий
  `accountStore.seatOf` ДО `resolveSeat`, чтобы отклонённый новичок не сжёг кресло),
  закрытое окно → 403; реконнект (ник уже держит место) не гейтится. Read-model:
  `MatchRegistry.entryOpen(matchId)` + `MatchMeta.entryWindowMs` + поля
  `entryOpen`/`entryClosesInMs` в `MatchSummary`; закрытая сессия выпадает из
  «Доступных», но остаётся в «Активных» у своих. Возраст = `state.time / timeScale`
  (реальное время работы мира: переживает рестарт, учитывает TIME_SCALE — на ×24
  4 реальных дня = 96 игровых). netserver: `ENTRY_WINDOW_MS` (деф. 4 дня, env),
  `admitNewSeat: (id) => registry.entryOpen(id)`, строка баннера. Тесты: 6 read-model
  (открыто/закрыто по timeScale, unknown=fail-secure, выпадение из «Доступных»,
  entryClosesInMs) + 3 транспортных (seat-lock и dev-ник: новичок 403, свой
  реконнект, обратная совместимость без окна). Живой смоук (окно=0 → новичок 403).
- **SES-2.4** ✅ `[proto]` **Лента сессий в главном меню.** Браузер матчей клиента
  (`#connect`, 3 вкладки Доступные/Активные/Архив над `GET /matches?nick=`) уже
  показывал карту/matchId, «День N», места X/Y, правила (×скорость, порог), статус
  и «Войти» кликом. SES-2.4 добавил статус **окна входа** (поля `entryOpen`/
  `entryClosesInMs` из read-model SES-2.3): на «Доступных» строка «вход ещё {Nд Mч}»
  (cyan; янтарь, если < суток) / «вход закрыт» (перечёркнуто) — закрытые сессии
  сервер и так не отдаёт в «Доступные», так что это подсказка «успеешь/торопись».
  `fmtJoinWindow` (дни/часы), sentinel `Number.MAX_SAFE_INTEGER` = без окна (старый
  сервер/dev) не рисует ничего. Реконнект своих в «Активных» окном не гейтится —
  там строки нет. RU/EN, CSS `.mwin`; контракт проверен живым `GET /matches`
  (`entryOpen:true`, `entryClosesInMs:345600000`=4 дня), i18n-гейт, чистый билд.
- **SES-2.5** ✅ `[srv]` `[proto]` **Регистрация/логин на игровом пути.** С
  `AUTH_JWT_SECRET` прото-хост включает ПОЛНЫЙ контур аккаунтов (тот же SE-1.x,
  что на боевом входе): `POST /auth/register|login` (scrypt, uniform-401 + decoy,
  per-IP лимит) → session-JWT (7 дней) → `GET /matches/:id/join` (Bearer) →
  join-токен (15 мин) → WS `?token=` — nick/ticket-рукопожатия ОТКЛОНЯЮТСЯ
  (токен — единственная identity). Join-роут повторяет гейт SES-2.3: неассигнящий
  `seatOf` ДО `resolveSeat`, закрытое окно → 403 `E_ENTRY_CLOSED` только для
  ПЕРВОГО входа логина; место принадлежит логину сессии (никем другим не зайдёшь),
  реджойн возвращает то же место. Без секрета — прежний nick+ticket (LAN, zero-setup);
  клиент самонастраивается по `GET /auth/status`. Клиент: поле «Пароль» на connect
  (показывается только в auth-режиме), zero-friction identity (login → 401 →
  register → 201; 409 = «неверный пароль»), session-JWT в localStorage per-server
  (пароль не хранится), реконнект сам минтит свежий join-токен (протухшая сессия →
  «введите пароль»). `users`-таблица в общей миграции (Memory/Postgres).
  `configFromEnv` экспортирован из index. Compose: `AUTH_JWT_SECRET` passthrough.
  Живой e2e (10 проверок): register→дубль 409→неверный 401→login→join без сессии
  401→join→WS token→welcome на своём месте→nick отклонён 401→реджойн то же место;
  плюс окно входа на auth-пути (window=0 → 403 E_ENTRY_CLOSED).
- **SES-2.6** ✅ `[srv]` `[ops]` **Плейтест-цикл ×24 — полный цикл живьём.**
  Compose: `TIME_SCALE` деф. **24** (1 реальный час = 1 игровой день; окна
  отсутствия/входа остаются РЕАЛЬНЫМИ — 3д/4д); для 24/7-постуры `TIME_SCALE=1`.
  Живой e2e полного цикла на одном сервере (AUTH+GATE+×24), 7/7: регистрация →
  session-JWT → лента (день/места/окно ~96ч) → join → место → WS `?token=` →
  gated `action.v1`-приказ → дельты → замер часов от первой к последней дельте
  ≈ ×24.0. Конец сессии: `checkEnd()` работает в ОБОИХ режимах (net-снапшоты
  несут `match`), `#endscreen` рендерит финал, сам конец матча закреплён
  victory-тестами ядра — живьём не переигрывается (победа занимает игровые
  недели даже на ×24).
