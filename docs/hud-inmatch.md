# Внутриматчевый HUD — адаптация (референс: Iron Order mobile)

> HUD боя для **PWA-клиента** (Stage 4), адаптированный из мобильного HUD жанрового
> референса (Iron Order, школа Bytro). Показывает **уже спроектированные** механики:
> день-счётчик (день-гейт), флоты/стеки, героев, ресурсы, валюты, боевые/движенческие
> действия. Рендерится как **framework-agnostic view-model** (паттерн `welcomeScreen.ts`)
> поверх **fog-проекции `visibleState`** — то есть HUD физически не может показать то, чего
> не видит игрок (туман уже вырезан на сервере). Тема — `theme.ts` (cyan-on-void glass).
> Клиент — **PWA-first** (`cross-platform-roadmap.md`; ремарка «RN» в `theme.ts` устарела).
> Парные доки: `main-menu.md` (мета-шелл НАД матчем), `state.md` (данные), `shields`/
> `ship-modules`/`squadrons`/`tech-tree`-роадмапы (стат-панель/иконки). Формат — кирпичики
> `HUD-*`. Зоны: `[cli]` (клиент) · `[core]`/`[srv]` где нужен новый данные/действие.

---

## 0. Что на референсе (инвентарь)

Сверху вниз на скриншоте Iron Order:

- **Стат-бар (верх):** флаг + название нации; ранг «8th» с прогрессом «66/826»; календарь
  «1» + «Next day in 23h 28min»; ряд из ~7 ресурсов (зерно/нефть/энергия/колба/слитки/каска/
  деньги) с числами.
- **Премиум-баннер:** золото «17 446» + зелёная «+» (магазин донат-валюты).
- **Карта:** изометрия провинции, лейн движения (пунктир к узлу), маркеры-стеки над юнитами
  («✈1», «🚶5»), 3D-модели выбранной бригады, вражеский юнит, ящик снабжения.
- **Панель действий (контекст выбранного юнита):** SPLIT · ADD ARMY · ATTACK · MOVE.
- **Панель выделения (низ):** флаг + «1ST INFANTRY BRIGADE (A 1)» + нация; pin/✕; строка
  «Idle | Speed: 45»; состав (лидер «5/10» + карточки стеков «3»/«2» + пустой слот с
  полосками силы); «❤ 61/61» + «+»; «Power Level: 117 | Damage Reduction: 15%».

## 1. Маппинг: Iron Order → Void Dominion

| Референс | Void Dominion | Источник данных (fog-проекция `visibleState`) |
|---|---|---|
| Флаг + «French Republic» | Эмблема игрока/корпорации + имя | `player.name` + корп-тег (мета); цвет = стартовый профиль домашнего узла |
| «8th» + бар 66/826 | Место в матче + прогресс очков | scoreboard `state.match.scores[me].total` → ранг; бар = до следующего места |
| 📅 «1» + «Next day in 23h28m» | **День сессии** + до следующего дня | `floor((state.time − startedAt)/MS_PER_DAY)`; отсчёт = `(граница дня − now)/timeScale` (реал-время). Тот же день, что гейтит технологии (`dayGate`) |
| Ряд ресурсов (7) | **5 ресурсов** + приход | `player.resources`: `credits`·`metal`·`food`·`energy`·`microelectronics`; rate — из эконом-формулы (offline-accrual) |
| Золото 17 446 + «+» | **Суверены** (донат) + кнопка магазина | мета-кошелёк (Суверены); опц. вторая пилюля — **Варранты** (аукцион) |
| «✈1»/«🚶5» над юнитами | Маркеры флотов (грубый бакет) | `visibleState.fleets` — свои точно; чужие по радару = **бакет размера по `signature`**, не точный состав |
| 3D-модели бригады | Рендер выбранного флота на узле | выбранный `fleet.units` (корабли) / `fleet.landing` (десант) |
| Оранжевый юнит | Вражеский флот (лишь если засечён) | `visibleState.fleets` (туман: `radarRange × signature`) |
| Ящик снабжения | Планета/узел с доходом · end-match дроп | планета-yield · дроп (economy) |
| **SPLIT** | Разделить флот | ⏳ новое действие `fleet.split` (нет в коде) |
| **ADD ARMY** | Слить флоты · загрузить десант | ⏳ `fleet.merge` (новое) · ✅ `army.load` (есть) |
| **ATTACK** | Штурм/бой | ✅ `fleet.assault` (двухфазный захват) · авто-бой на контакте |
| **MOVE** | Приказ по лейнам | ✅ `fleet.move` (movement-модуль, маршрут по `route.ts`) |
| «1ST INFANTRY BRIGADE» + нация | Имя флота + владелец | `fleet.id`/имя + `owner` |
| 📌 / ✕ | Следить (центрировать/трек) / снять выделение | UI-аффорданс |
| «Idle» / «Speed: 45» | Статус флота + скорость | `fleet.movement`(null=Idle · есть=В пути · `edge`=На лейне · `battleId`=В бою) + `fleetBaseSpeed` (медленнейший корабль) |
| Лидер «5/10» + карточки «3»/«2» | **Командир-герой** (адмирал) + стеки кораблей | герой на узле флота (`state.heroes` по `owner`+`location`) + `fleet.units`/`landing` (count + condition-бар = `hp`/пул) |
| «❤ 61/61» + «+» | **HP флота** (корпус[+щит]) + ремонт | Σ count×hp; со **щитами** (SH-роадмап) — две полоски hull/shield; «+» = порт/дроны/ремонт-модуль |
| «Power Level: 117» | Боевая мощь флота | производная: стеки×статы + герой + тех + модули (эффективные статы, `ship-modules`) |
| «Damage Reduction: 15%» | Защитный бонус | производная: герой-абилка / тех / щит / броня-модуль |

**Ключевой вывод:** структура референса ложится на VD почти 1:1, потому что домен-концепты
совпадают (мобильные армии → флоты; провинции → секторы-узлы; лейны → звёздные лейны;
день-таймер → наш день-гейт; офицер → герой-адмирал; HP → корпус+щит). Отличия — доменные:
космос вместо WWII, узлы-лейны вместо непрерывной карты, орбита/десант вместо линии фронта.

## 2. Раскладка HUD (регионы, мобиль-first)

```
┌─────────────────────────────────────────────────────┐
│ [эмблема] Имя · корп     🏅 4-е  📅 День 1 · 23:28   │  ← A. Стат-бар (тонкий, фикс. верх)
│ ◈ credits metal food energy micro   ◆ Суверены  +   │
├─────────────────────────────────────────────────────┤
│                                                     │
│                  B. КАРТА (секторы/лейны/флоты)     │  ← жест-зона: пан/зум/тап-выделение
│                     маркеры по fog-проекции         │
│                                                     │
├─────────────────────────────────────────────────────┤
│      ⑃ Разделить   ＋ Слить   ⚡ Атака   ↳ Движение   │  ← C. Контекст-действия (над панелью)
├─────────────────────────────────────────────────────┤
│ [флаг] 1-й Ударный флот (A1) · Имя        📌  ✕     │  ← D. Панель выделения
│ 📍 В пути  »  Скорость 45                            │
│ [адмирал 5/10] [🚀×3 ▮▮▯] [🛰×2 ▮▮▮] [＋]  ❤ 61/61 ＋ │
│ Мощь: 117            Сниж. урона: 15%                │
└─────────────────────────────────────────────────────┘
```

- **A. Стат-бар** — фикс. верх, две строки, safe-area-aware. Идентичность + ранг + **день-таймер**
  (главный async-якорь) слева-направо; ресурсы + валюта во второй строке. Тап по ресурсу →
  разбивка прихода/расхода; тап по «+» валюты → магазин (мета).
- **B. Карта** — основная жест-зона (пан/зум одной рукой; тап = выделить флот/планету).
  Рисует **только `visibleState`**: свои флоты точно, чужие — бакет по радару. Лейны движения,
  орбита, бой-маркеры.
- **C. Контекст-действия** — появляются при выделенном флоте; **крупные тач-таргеты** (≥48dp),
  доминантная кнопка (Атака) — акцент-цветом (`theme.red`). Дизейбл с причиной (напр. «нет
  цели в радиусе»).
- **D. Панель выделения** — «карточка юнита» из референса: имя+владелец, статус+скорость,
  состав (командир-герой + стеки-карточки с condition-барами), HP (корпус/щит) + ремонт,
  производные (Мощь / Сниж. урона). Свайп-вниз/✕ — свернуть.

## 3. View-model (framework-agnostic, паттерн `welcomeScreen.ts`)

Чистый view-model + редьюсер тапа-в-намерение — рендер-агностично, JSON-сериализуемо,
детерминированно, **строится из `visibleState`** (значит fog-safe by construction):

```ts
export interface HudModel {
  topBar: {
    identity: { name: string; corpTag?: string; color: string };
    rank: { place: number; score: number; nextScore?: number };   // scoreboard
    day: { n: number; nextInMs: number };                          // из state.time/startedAt/timeScale
    resources: ResourcePill[];                                     // 5 шт.
    wallet: { sovereigns: number; warrants?: number };             // мета-кошелёк
  };
  selection?: FleetView;        // выставлен при выделенном флоте
  actions: HudAction[];         // контекст-действия для selection
}
export interface ResourcePill { id: ResourceId; amount: number; ratePerDay?: number }
export interface FleetView {
  id: string; name: string; owner: string;
  status: 'idle' | 'moving' | 'parked' | 'battle';
  speed: number;                                   // fleetBaseSpeed
  commander?: { name: string; level: number };     // герой на узле флота
  stacks: StackCard[];                             // fleet.units (+ landing)
  hull: { cur: number; max: number };
  shield?: { cur: number; max: number };           // ✅ реально (shields SH-0.1/0.2)
  power: number; damageReduction: number;          // производные
}
export interface StackCard { unit: string; count: number; condition: number /* 0..1 = hp-доля */ }
export interface HudAction { id: 'split' | 'merge' | 'attack' | 'move'; enabled: boolean; reasonKey?: string }

/** Pure: fog-проекция + выделение → HUD-модель. Никаких Date/random; i18n — ключами. */
export function createHudModel(view: VisibleState, viewerId: string, selectedFleetId?: string): HudModel;
/** Pure: тап действия → серверное намерение (или {ok:false, code}). Как resolveWelcomeAction. */
export function resolveHudAction(action: HudTap, model: HudModel): HudIntent | { ok: false; code: string };
```

Строки — через bundle (i18n-шов, `main-menu.md §5.4`); иконки/цвета — `theme.ts`.

**Реализовано — HUD-1a, зоны A + D** (`packages/client/src/matchHud.ts`, паттерн `welcomeScreen.ts`).
Объём был «только A и D» (карта B и действия C не трогались), поэтому вместо одного `createHudModel`
зоны вышли **двумя сфокусированными фабриками**, каждое поле — из реального `visibleState`:

- `createStatusBarModel(state, viewerId, data?)` → зона A. `commander`=`player.name`;
  `faction`=`player.faction` (контент-id, **не** корп-тег — такого поля в рантайме нет);
  `rank`/`players` из `match.scores[*].total` (туман не режет `match`; тай-брейк по id —
  детерминизм); `day`/`dayTimeMs` = `floor((time−startedAt)/MS_PER_DAY)` (та же свёртка, что у
  match-browser; `elapsed` клампится в ≥0 — оба поля когерентны при обратном skew); `resources` в
  каноническом порядке `data.resources` (отсутствующий → 0); `defeated`.
- `createSelectionModel(state, fleetId, viewerId, data?)` → зона D для **флота**: `status`
  (`transit`/`parked`/`stationed`) + пункт/ETA из `movement`/`edge`/`location`; `ships`=`fleet.units`;
  `hull`=`Σ count×def.stats.hp` (текущее — из боевого пула `stack.hp`); `commander` = герой флота
  (`name`+`grade`, **без** числового уровня; страж `hero.owner===viewerId` — чужой командир не течёт
  даже на не-туманном состоянии); `inCombat`, `mine`, владелец.

Обе фабрики **fail-secure** (`E_NO_PLAYER` / `E_NO_SELECTION`) и мягко деградируют без `data`
(нет `hull`/`domain`). **`shield?: {current,max}`** — уже реальный (shields-roadmap SH-0.1/0.2:
стат `shield` + пул `shieldHp`); опускается для беcщитового флота (одна полоска). **Осознанно НЕ
эмитим** (нет опоры в ядре — не выдумываем числа): `power`, `damageReduction`, числовой уровень
командира, кошелёк Суверенов, ранг-бар «66/826», `ratePerDay`. Появятся, когда механика ляжет в код
(`ship-modules`-роадмап, HUD-2). Объединённый `createHudModel`/`resolveHudAction` (зоны B/C + мета) —
следующий кирпич.

## 4. Мобильные ограничения / тема

- **Тач-таргеты ≥ 48dp**, действия — в нижней трети (зона большого пальца); стат-бар/карта —
  вне частых тапов. Safe-area (вырез/навбар) — паддинги.
- **Тема** — токены `theme.ts` (cyan `#35d6e6`, red `#ff5a4d` для Атаки/опасности, amber для
  transient-статуса, glass-панели `rgba(3,14,18,.82)`). Один TS-движок → один вид на web/TWA/iOS.
- **Плотность** — референс плотный; на VD ужимаем ресурсы до 5 и прячем вторичное за тапом
  (разбивки/тултипы), чтобы не перегружать узкий экран.

## 5. Инварианты (нарушение = баг)

- **Fog-safe by construction:** HUD строится из `visibleState` (сервер уже вырезал чужое —
  ресурсы/технологии/**учёного**/скрытые флоты). Клиент не может отрисовать то, чего не видит.
- **Server-authority:** тап действия → **намерение** на сервер (`fleet.move`/`assault`/…),
  никогда не мутирует состояние локально; предиктивный рендер — поверх, откатываемо.
- **Чистый view-model:** без Date/random, JSON-сериализуемо, fail-secure (кривой тап →
  стабильный код). Тестируется как `welcomeScreen.ts`.
- **i18n + data-driven:** строки ключами; ресурсы/иконки/действия — из данных/темы, не хардкод.
- **Graceful:** нет выделения → нет панели D/действий; нет щитов в коде → одна HP-полоска;
  нет героя на узле → слот командира пуст.

## Последовательность

`HUD-1` (view-model на готовых данных — можно строить сейчас) → `HUD-2` (производные Мощь/
Сниж.урона) → `HUD-3` (split/merge как core-действия) → `HUD-4` (щит-полоска/иконки модулей по
мере готовности боевого контента). Мета-пилюли (ранг/валюта/магазин) подключаются к мета-слою
отдельно.
