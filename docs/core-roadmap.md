## Фаза 0 · Версияирование сейвов и реплей (🔴 из gap-анализа)

### CR-0.1 · Версия данных/правил per-match + миграции `[core][data]` ⏳ — L

**Цель:** обновление ядра/данных не ломает идущие неделями матчи.
**Подзадачи:** зафиксировать `version` в `GameState` (есть pin) → реестр миграторов «формат N→N+1»; матч помнит свою версию; миграция при загрузке (ленивая) либо матч доигрывается на старых правилах; тест round-trip миграции.
**Готово, когда:** матч версии N загружается и продолжается после апдейта без потери прогресса.

### CR-0.2 · Детерминированный реплей-тулинг `[core]` ⏳ — M

**Цель:** воспроизводить баг/инцидент из (seed + лог действий) — ядро это позволяет «даром».
**Подзадачи:** формат реплея (начальный seed + упорядоченные действия + ctx.now); раннер «прогнать до кадра K»; CLI для дебага; хранилище реплеев — в `persistence-roadmap.md`.
**Готово, когда:** из реплея байт-в-байт воспроизводится конечное состояние (golden).

### CR-0.3 · Детекция десинхрона (хеш состояния) `[core][srv]` 🚧 — M

**Цель:** «детерминизм» — гарантия, а не вера: ловить расхождение клиент↔сервер в рантайме.
**Подзадачи:** ✅ канонический детерминированный **хеш состояния** (`hashState`, `state/hash.ts` — ключи сортируются, поэтому серверное и дельта-восстановленное на клиенте состояния хешируются одинаково; golden + дельта-round-trip тесты); ⏳ периодическая сверка сервер↔клиент (или shadow-mode) 🔒(сервер); расхождение → форс-ресинк (полный снапшот) + алёрт. **(= `sprint-1.md` S1.6.)**
**Готово, когда:** ✅ искусственный десинк ловится сверкой хешей (`hash.test.ts`); ⏳ в рантайме поднимается алёрт (нужен сервер).
